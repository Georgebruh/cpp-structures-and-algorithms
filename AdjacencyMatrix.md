# Adjacency Matrix 

### 1. Overview of AdjacencyMatrix
The `AdjacencyMatrix` is a concrete data structure that implements a Graph. It utilizes a two-dimensional grid (a matrix) of boolean values to represent connections between different vertices, making it incredibly fast for checking if a direct relationship exists between any two specific points.

### 2. Architectural Components
The `AdjacencyMatrix` strictly follows a predefined interface to ensure it integrates seamlessly with any other graph-based algorithms in the repository, while managing its own internal grid memory.

#### A. The Graph Interface ([`graph.hpp`](./Interfaces/graph.hpp))
`AdjacencyMatrix` inherits from a base interface class called `Graph`. This interface dictates that any conforming graph structure must implement the following core methods:
* **`addEdge(int i, int j)`** and **`removeEdge(int i, int j)`**: Creates or destroys a directed connection from vertex `i` to vertex `j`.
* **`hasEdge(int i, int j)`**: Returns a boolean confirming if a specific connection currently exists.
* **`outEdges(int i)`** and **`inEdges(int i)`**: Returns a collection of all vertices connected from or to a specific point.
* **`getNumVertices()`**: Returns the total number of vertices in the graph.

#### B. Raw Memory Management
Instead of relying on nested standard vectors, the implementation utilizes a raw dynamically allocated two-dimensional array (`bool** matrix`) to manage memory manually for maximum lookup speed.
* **Allocation & Cleanup:** The constructor takes the total number of vertices and allocates a primary array of pointers on the heap. It then loops through and allocates a secondary boolean array for each row, initializing every single cell to `false`. To prevent massive memory leaks, the destructor carefully loops through and deletes each individual row before safely deleting the primary array.

### 3. Deep Dive into `AdjacencyMatrix` Logic ([`adjacency_matrix.cpp`](./Implementations/adjacency_matrix.cpp))
The `AdjacencyMatrix` class maintains two critical variables: an integer `numVertices` to track the strict boundaries of the graph, and the `matrix` grid itself.

#### Edge Operations: `addEdge`, `removeEdge`, and `hasEdge`
Because the matrix is a perfect grid, interacting with specific edges requires zero searching.
* **Instant Lookups:** To check if an edge exists, or to add or remove one, the structure performs a quick bounds check to ensure the vertices are valid. It then directly accesses the specific row `i` and column `j` (`matrix[i][j]`) and updates or reads the boolean flag. This guarantees an ultra-fast $O(1)$ time complexity for direct edge manipulation.

#### Vertex Queries: `outEdges` and `inEdges`
To find all connections tied to a specific vertex, the matrix must scan its internal grid.
* **`outEdges(int i)`:** This method gathers all outgoing connections by looking at row `i`. It iterates through every column in that single row; if it finds a `true` value at column `j`, it adds `j` to the results list.
* **`inEdges(int i)`:** This method gathers all incoming connections by looking at column `i`. It iterates down through every single row; if it finds a `true` value in row `j` at the target column, it adds `j` to the results list. Both of these operations scale linearly based on the total number of vertices, taking $O(V)$ time.

### 4. Performance Testing and Benchmarking
To validate the efficiency of the matrix while respecting computer memory limits, the project includes a specialized graph benchmarking suite.

* **The C++ Benchmark ([`benchmark.cpp`](./Benchmarking/benchmark.cpp)):** The `benchmarkAdjacencyMatrix` function tests how quickly the structure can generate sequential connections. It loops through $N$ elements, starting from 1,000 up to a strict cap of 20,000 in increments of 1,000. (The cap prevents the $N \times N$ matrix from consuming excessive gigabytes of RAM). For each step, it records the exact time it takes to add $N-1$ edges using `std::chrono::high_resolution_clock`.
* **Live Data Visualization ([`live_graph.py`](./Benchmarking/live_graph.py)):** The data generated by the C++ executable is piped into the Python script. Because the maximum element count is carefully restricted to 20,000, the Python visualizer contains specific logic (`ax.set_xlim(0, 22000)`) to dynamically zoom the X-axis. This ensures the resulting performance curve is perfectly framed and readable.
